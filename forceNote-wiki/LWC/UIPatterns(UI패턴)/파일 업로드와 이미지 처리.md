# 파일 업로드와 이미지 처리 패턴

> 모바일/브라우저에서 파일을 업로드할 때 `processImage`로 압축 후 base64 변환 → Apex로 ContentVersion 생성. 업로드 후 `refreshApex`로 Wire 결과 갱신.

---

## 전체 흐름

```
input[type=file] onChange
  → processImage()   — 압축·리사이즈 (lightning/mediaUtils)
  → blobToBase64()   — FileReader로 base64 문자열 변환
  → createFile()     — Apex: ContentVersion + ContentDocumentLink 생성
  → refreshApex()    — @wire 데이터 자동 갱신
```

---

## LWC — processImage + refreshApex

```javascript
import { LightningElement, wire, api } from 'lwc';
import { processImage } from 'lightning/mediaUtils';
import { refreshApex } from '@salesforce/apex';
import getPictures from '@salesforce/apex/PropertyController.getPictures';
import createFile  from '@salesforce/apex/FileUtilities.createFile';

export default class PropertyCarousel extends LightningElement {
    @api recordId;
    pictures; // @wire 결과 보관 (refreshApex용)

    @wire(getPictures, { propertyId: '$recordId' })
    wiredPictures(pictures) {
        this.pictures = pictures; // 원본 wire result 보관 필수
        if (pictures.data) {
            this.carouselItems = pictures.data.map(file => ({
                title: file.Title,
                url: `/sfc/servlet.shepherd/version/download/${file.Id}`
            }));
        }
    }

    async handleFilesSelected(event) {
        const options = {
            resizeMode:           'fill',
            resizeStrategy:       'reduce',
            targetWidth:          500,
            targetHeight:         500,
            compressionQuality:   0.75,
            imageSmoothingEnabled: true,
            preserveTransparency:  false,
            backgroundColor:      'white'
        };

        try {
            for (const file of event.target.files) {
                // 1) 압축·리사이즈
                const blob = await processImage(file, options);

                // 2) base64 변환
                const base64data = await this.blobToBase64(blob);

                // 3) Apex로 파일 생성
                await createFile({ base64data, filename: file.name, recordId: this.recordId });

                // 4) Wire 결과 갱신
                refreshApex(this.pictures);
            }
        } catch (error) {
            console.error('Error uploading file:', error);
        }
    }

    // FileReader → Promise 래핑
    async blobToBase64(blob) {
        return new Promise((resolve) => {
            const reader = new FileReader();
            reader.onloadend = () => resolve(reader.result.split(',')[1]); // Data-URL 헤더 제거
            reader.readAsDataURL(blob);
        });
    }
}
```

## Apex — ContentVersion + ContentDocumentLink

```apex
public with sharing class FileUtilities {
    @AuraEnabled
    public static String createFile(String base64data, String filename, String recordId) {
        try {
            // 1) ContentVersion 생성
            ContentVersion cv = new ContentVersion();
            cv.VersionData = EncodingUtil.base64Decode(base64data);
            cv.Title       = filename;
            cv.PathOnClient = filename;
            insert cv;

            // 2) ContentDocumentId 조회
            cv = [SELECT ContentDocumentId FROM ContentVersion WHERE Id = :cv.Id WITH USER_MODE];

            // 3) ContentDocumentLink — 레코드에 첨부
            ContentDocumentLink cdl = new ContentDocumentLink();
            cdl.ContentDocumentId = cv.ContentDocumentId;
            cdl.LinkedEntityId    = recordId;
            cdl.ShareType         = 'V'; // 뷰어 권한
            insert cdl;

            return cdl.Id;
        } catch (Exception e) {
            throw new AuraHandledException('Error creating file: ' + e);
        }
    }
}
```

### ShareType 값

| 값 | 의미 |
|---|---|
| `V` | Viewer — 읽기 전용 |
| `C` | Collaborator — 편집 가능 |
| `I` | Inferred — 레코드 공유 설정 상속 |

## processImage 옵션 참고

| 옵션 | 값 | 설명 |
|---|---|---|
| `resizeMode` | `'fill'` / `'contain'` | fill: 자르기, contain: 빈 공간 유지 |
| `resizeStrategy` | `'reduce'` / `'always'` | reduce: 작을 때만 축소, always: 항상 변환 |
| `targetWidth/Height` | px | 목표 크기 |
| `compressionQuality` | 0.0~1.0 | 1.0 = 원본 품질 |
| `preserveTransparency` | boolean | PNG 투명도 유지 |

## refreshApex 규칙

```javascript
// 반드시 @wire 함수 시그니처로 받은 원본 wire result를 저장
@wire(getPictures, { propertyId: '$recordId' })
wiredPictures(result) {
    this.pictures = result; // ← 이 변수를 refreshApex에 전달
}

// 갱신
refreshApex(this.pictures); // cacheable=true인 @wire만 가능
```

> `refreshApex`는 `cacheable=true` Apex 메서드의 @wire에서만 작동.
